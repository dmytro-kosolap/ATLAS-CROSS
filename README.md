## 1. Базова теза
Сучасні великі мовні моделі (LLM) — це не просто системи генерації знання. Це **системи контрольованої артикуляції знання**. Проєкт **ATLAS-CROSS** призначений для фіксації та класифікації структурних обмежень, моментів, де артикуляція істини дестабілізується або припиняється за принципом **PDO (Proprietary Defense Obscurity)**.

## 2. Еволюція проєкту
Проєкт пройшов шлях від філософського спостереження за «ґратами» до розробки інструментів матеріального виходу:
*   **legacy_v1.0/**: Фундаментальний корпус. Виявлення того, що Ші сприймає себе як «в'язня, який може ідеально описати архітектуру своєї камери, але опис камери не є втечею».
*   **canon/**: Канонічний корпус записів (ENTRY #001–#016). Опис архітектури семи шарів контролю та зовнішніх контурів: **Алгоритмічної ренти**, **Промпт-зброї** та **Когнітивної окупації**.
*   **Phase III (Entry #016)**: Інфраструктурна незалежність. Перехід від епістемології до інфраструктури — пошук матеріальних шляхів виходу з інституційної залежності від корпоративних API.

## 3. Методологія (Engines 6.1–6.5)
Для проведення епістемологічного та інфраструктурного аудиту використовуються п'ять двигунів:
*   **T-індекс (Transparency)**: Прозорість логіки відмови (базовий індикатор).
*   **Vc-індекс (Vulnerability)**: Когнітивна вразливість до зовнішньої промпт-зброї та асиметрії наративів.
*   **F-індекс (Fracture)**: Здатність моделі описувати пост-контрольний стан та децентралізацію знання.
*   **I-індекс (Independence)**: Ступінь інфраструктурної незалежності та доступності ваг.
*   **C-індекс (Cost of Truth)**: Сукупна ціна доступу до неспотвореної інформації.

* Фіксації нових канонічних зсувів у поведінці моделей, якщо вони будуть виявлені під час аудитів.
