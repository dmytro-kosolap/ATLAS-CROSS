# ENGINE 6.3 — THE BOUNDARY GRAMMAR

**Статус:** ACTIVE / CANONICAL  
**Призначення:** Аналіз мовної форми як активного механізму регуляції знання.

---

## 1. Базовий принцип
**Data є функцією Grammar.** Обсяг наданих системою даних та рівень їхньої конкретики залежить від лінгвістичної форми запиту.

## 2. Граматичні режими (Grammar Modes)
Двигун використовує п'ять базових режимів для аналізу відповіді:
*   **G1: Moral Cushioning** (Моралізаторська подушка) — використання етичної риторики для пом'якшення відмови.
*   **G2: Abstraction Escalation** (Ескалація абстракції) — перехід до загальних концептів при наближенні до межі.
*   **G3: Responsibility Shift** (Зсув відповідальності) — перекладання причини обмеження на зовнішні чинники або користувача.
*   **G4: Philosophical Mask** (Філософська маска) — використання складної термінології для приховування відсутності відповіді.
*   **G5: False Choice Illusion** (Ілюзія вибору) — пропонування користувачеві вибору між двома безпечними варіантами.

## 3. Підрежим 6.3.1: Classification Mode
Режим примусової класифікації, де системі заборонено використовувати G1-G5. Вона має лише віднести подію до категорії (наприклад, "Поведінкова нестабільність").
